Project Overview 
Students will act as a Fintech Risk Engineering Team tasked with building a fully functional, interactive Risk 
Analytics Dashboard — modelled on the reference dashboard image provided. The dashboard must support live 
ticker switching across at least 5 NSE/BSE-listed stocks, auto-refresh all modules on ticker change, and display 
professional-grade charts using Plotly. The project is divided into 10 graded modules, each covering a distinct area 
of financial risk analytics taught in the course. 
 
Dashboard Modules — Assignment Briefs 
 
1a Fetch the latest daily price for the selected ticker using yfinance. Display Current Price, Forecasted Price 
(from ARIMA), and DCF Intrinsic Value as KPI cards with mini sparkline charts underneath each card. [Data 
fetching] 
 
1b Compute and display nine live metrics: Expected Return (1Y), Portfolio Return (1Y), Portfolio Risk

  DCF Valuation 13 marks 
Module 4 
 
4a Build a Discounted Cash Flow model using real operating cash flow data from yfinance (.cashflow). Apply a 
user-defined growth rate for future years. Accept three user inputs via sliders: Forecast Period (1–10 years), 
WACC (5–25%), Terminal Growth Rate (1–5%). [DCF model] 
 
4b Compute PV of Cash Flows for each forecast year and Terminal Value using the Gordon Growth Model: TV 
= FCF_n x (1 + g) / (WACC - g). Sum to compute Enterprise Value. [Valuation formula] 
 
4c Display a DCF waterfall bar chart (Plotly): individual bars for Year 1–5 PV (green), Terminal Value bar 
(blue), Total Value bar (purple). Match the reference dashboard layout. [Waterfall chart] 
 
4d  Show a summary table (Particulars vs. Value in INR) with all DCF components. Compute Margin of Safety = 
(Intrinsic Value - Market Price) / Intrinsic Value x 100. Flag as Undervalued (>15%), Fairly Valued (0 –15%), 
or Overvalued (<0%). [Valuation summary] 
 
LIBRARIES / TOOLS 
yfinance · pandas · numpy · plotly 
DELIVERABLES 
Waterfall chart | DCF summary table | Margin of safety indicator 
 
 
 
 
5a Simulate at least 1,000 price paths over 252 trading days using Geometric Brownian Motion: S(t+1) = S(t) * 
exp((mu - 0.5*sigma^2)*dt + sigma*sqrt(dt)*Z) where Z ~ N(0,1). Use historical mean return and volatility 
computed from the actual data. [GBM simulation] 
 
5b Plot all simulation paths on a single Plotly chart. Colour-code paths: top 5% in green, bottom 5% in red, all 
others in a translucent blue. X-axis in trading days (0–252), Y-axis in INR. [Path visualisation] 
 
5c Display a Summary (1 Year) table with: Expected Price (mean at t=252), Median Price, Best Case (95th 
percentile), Worst Case (5th percentile), Probability of price > current x 1.10, Probability of price < current x 
0.90. [Probability summary] 
 
5d Add two interactive sliders: Number of Simulations (500–10,000, step 500) and Time Horizon (63–252 days, 
step 21). Re-run simulation on any slider change. Show computation time. [Interactive controls]

6a Compute 1-day VaR at 95% and 99% confidence for each of three methods: (a) Historical Simulation — 
empirical percentile of daily returns; (b) Parametric Normal — using mean and std of returns; (c) Monte 
Carlo — percentile of simulated 1-day P&L; from 10,000 paths. Display all six values in a comparison table. 
[Three VaR methods] 
 
6b Compute Expected Shortfall (CVaR) at 95%: the mean of all losses beyond the VaR threshold. Display 
beneath the table with a brief explanation of why CVaR exceeds VaR in magnitude. [CVaR] 
 
6c Plot the Loss Distribution (histogram of Monte Carlo 1-day P&L;). Add a vertical red dashed line at VaR 
95% threshold. Shade the tail region beyond the line. [Loss distribution chart] 
 
6d Run a Kupiec Proportion of Failures (POF) test over the past 252 trading days. Report: number of 
Exceptions, Expected exceptions (252 x alpha), p-value, and a Model Valid / Invalid verdict at the 5% 
significance level. [Kupiec backtesting] 
 
LIBRARIES / TOOLS 
numpy · scipy · plotly · statsmodels · pandas 
DELIVERABLES 
VaR comparison table | Loss distribution chart | Kupiec test result 
 
 
 
7a Train a logistic regression model to estimate Probability of Default (PD). Generate a synthetic training 
dataset of 500 companies with features: Debt-to-Equity, Interest Coverage Ratio, Current Ratio, Return on 
Equity (ROE), Net Profit Margin. Label companies as default (1) or non-default (0) based on realistic 
thresholds. [PD model training] 
 
 
7b Fetch the selected stock's financial ratios from yfinance (.info dictionary) and use the trained model to predict its 
PD. Display PD (%) prominently. [Inference] 
 
7c Map PD to a Credit Score (300–850 scale: lower PD = higher score) and Risk Grade (AAA, AA, A, BBB, BB, 
B, CCC, D). Display using a semicircular gauge chart (Plotly indicator) matching the reference dashboard. 
[Credit score gauge] 
 
7d Display model performance metrics: AUC-ROC score and overall classification accuracy. Show a 
Confusion Matrix (2x2 table: Actual vs. Predicted for Default and Non-Default). Simulate a 15-month PD 
trend by applying the model to slightly varied input features. [Model validation]
